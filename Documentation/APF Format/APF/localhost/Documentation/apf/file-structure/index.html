File Structure

Structure

APF{
    byte[4] magicNumbers (ASCII .apf)
    byte[2] fileVersion
    String[1] appName
    String[1] appIdentifier
    byte[1] appMajorVersion
    byte[1] appMinorVersion
    byte[1] appIconCount
    AppIcon[appIconCount] appIcons
    byte[1] splashImageCount
    SplashImage[splashImageCount] splashImages
    byte[4] binaryLength
    byte[binaryLength] binaryData
    byte[2] fileCount
    File[fileCount] files
}

String{
    byte[1] stringLength
    byte[stringLength] string
}

AppIcon{
    byte[2] iconSize
    byte[4] dataLength
    byte[dataLength] data
}

SplashImage{
    byte[2] imageWidth
    byte[2] imageHeight
    byte[4] dataLength
    byte[dataLength] data
}

File{
    String[1] filename
    byte[1] subDirCount
    String[subDirCount] subDirs
    byte[4] dataLength
    byte[dataLength] data
}

Detail

APF

magicNumbers

A sequence of 4 bytes (.apf in ASCII) that signals the start of an application package file. This makes the file independent from the file extension and alows to filter out files that have the .apf extension by accident

fileVersion

2 Bytes indicating the file version. Used for backward compatibility

appName

The applications display name. This field contains the name you want to display on devices. Systems are not required to display this name though. But its encouraged as it is more readable than the appIdentifier

appIdentifier

The app identifier is much like Androids identifier: It has to be unique and normaly follows the same naming convention (reverse URL com.companyname.productname)

majorVersion

The apps major version should be increased on big feature or layout changes. A single byte

minorVersion

The apps minor version should be increased on bugfixes or minor changes. A single byte

appIconCount

Number of icons supplied with this package file. There has to be at least 1 icon and at most 255.

appIcons

List of icons for this application. Icons can contain transparency. They have to be square and every icon has to have a different size

splashImageCount

Number of splash images supplied with this package. There has to be at least 1 splash icon and at most 255.

splashImages

List of splash images for this applications. Splash images can be shown when the application starts or as background image. Splash images cannot contain transparency and every splash image has to have a different size

binaryLength

Length in bytes of the binary file

binaryData

Data of the binary file. The binary name is not included as the file will be renamed on the target device

fileCount

Number of files within this package. There can be up to 65535 files included.

files

List of files. These files will be extracted on the target device into a folder the application has readonly access to. On application update these files will be deleted and replaced with the file entries in the updated package

 

String

stringLength

Number of bytes that represent the string

string

String content. Usually UTF8

 

AppIcon

iconSize

Width and height of the icon. As app icons have to be square, there is only one value for both.

dataLength

Size of the icon in bytes

data

Bytes that form the app icon file. Usually should be png, but others may be supported too

 

Splash Image

imageWidth

Width of the splash image

imageHeight

Height of the image

dataLength

Size of the image in bytes

data

Bytes that form the splash image file. Usually should be png, but others may be supported too

 

File

filename

The files name (including extension if any)

subDirCount

Number of subdirectories for this file

subDirs

List of subdirectories that form the application path. These directories will be created in sequence before placing the file within the last directory

dataLength

Size of the file in bytes

data

Byte data of the file
